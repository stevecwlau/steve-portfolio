Project information

  • Title: Polygon extraction of urban area in HK
  • Data source: The Copernicus Open Access Hub
  • Software used: QGIS, RStudio
  • Skills: LULC, fish net grids, raster processing, zonal statistics, polygon dissolve, polygon smoothing

The collection of satellite images is the very first step of this project. A S2MSI2A sentinel-2 imagery with cloud cover percentage less than 2 that covers the entire administrative boundary of Hong Kong was downloaded from the Copernicus Open Access Hub. Once downloaded, I used the Blue (B2), Green (B3), Red (B4), Near infrared (B8) and Short-wave infrared (B11) bands to form a stacked GeoTIFF file. After the bands were stacked up, it was then plugged into R as one of the parameters along with one set of training data and another set of validation data. The ratio for training set and validation set for this particular analysis is 70-30.

The first stage of the script involved two indices calculations of Normalized Difference Vegetation Index (NDVI) and Normalized Difference Water Index (NDWI) which were then added on top of the stacked GeoTIFF to form a new raster brick of covariates. After that, the script called for the training and validation sets to assign code values to raster cells where the vector and raster layers overlap. This would then create a new training dataset through the combination of classes layer and raster brick of training pixels. All values were then extracted into a matrix to form part of the training model. Finally, the newly formed raster brick of covariates was classified using the SVM model within R environment. The resulting classification output has a percentage yield of almost 83% accuracy. As shown in picture 3, the final output of the LULC classification consists of 7 classes: agriculture, bareland, forest, grassland, urban, urban shadow and water. Urban and urban shadow were then extracted to form the urban polgyons using various geoa stacked GeoTIFF file processing tools in QGIS (see picture 4).
